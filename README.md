
# ☢️ SPECT/CT 기반 3D 병변 자동 분할 및 선량 평가 대시보드 (Clinical Validation Guide)

안녕하십니까, 핵의학과 및 내분비내과 의료진 여러분.
본 Jupyter Notebook 대시보드는 **갑상선기능항진증 환자의 I-131 맞춤형 3차원 정밀 흡수선량(Gy) 평가**를 위한 기하학적 입력값(치료 용적)을 추출하는 핵심 도구입니다.

기존에는 임상 현장에서 경험에 의존하거나 수작업으로 Target Volume(ROI)을 설정해야 하는 엄청난 업무 부하가 있었습니다. 이 툴은 최신 컴퓨터 비전 알고리즘(FBP, OSEM, MAP)을 통해 객관적인 분할 체적(ml)과 ROI 윤곽선을 자동 산출합니다.

선생님들께서는 본 대시보드를 통해 **자동화된 병변 윤곽선이 실제 임상적 소견과 얼마나 정확히 일치하는지 시각적으로 검증(Validation)**해 주시면 됩니다. 여기서 확정된 정밀한 3D 체적 데이터는 물리학팀의 몬테카를로(GATE/Geant4) 물리 엔진으로 연동되어, 기존 2D Planar 영상의 한계를 극복한 가장 정밀한 환자 맞춤형 선량 평가 결과를 제공하게 될 것입니다.

## 🚀 2. 핵심 대시보드 실행 가이드 (Cell 2 & Cell 5)

전체 셀을 순서대로 실행(Shift + Enter)하시면 됩니다. 가장 중요한 임상적 검증 툴은 **Cell 2(다중 단면 뷰어)**와 **Cell 5(최종 대시보드)**에 위치해 있습니다.

데이터를 불러오고 슬라이스를 재구성하는 데 약간의 시간(Progress Bar 진행)이 소요되며, "✅ Engine Ready." 문구가 뜨면 하단에 조작 가능한 슬라이더 UI가 나타납니다.

### 📌 Cell 2: 다중 단면 비교 대시보드 (2x4 Matrix)

이 셀은 선생님들께서 주로 보시는 MIP(Maximum Intensity Projection) 스카웃 영상과 함께, 각 알고리즘별 성능을 한눈에 비교할 수 있도록 설계되었습니다.

* **관전 포인트:** FBP 알고리즘의 노이즈, 임상 표준인 OSEM의 결과, 그리고 노이즈를 억제하면서 경계선을 날카롭게 보존하는 **MAP(베이지안) 알고리즘**의 차이를 눈으로 직접 비교해 보십시오.
* **활용법:** 하단의 `Threshold %` 슬라이더를 조절하여, 병변 주변부의 Background Activity가 배제되고 실제 Target Volume만 깔끔하게 윤곽선(Lime color)으로 잡히는 최적의 역치값을 찾아주시면 됩니다.

### 📌 Cell 5: 임상 검증용 최종 대시보드 (White Background)

가장 직관적으로 병변을 탐색하고 논문/학회 발표용(PPT) 자료를 캡처하기 좋도록 배경을 흰색으로 최적화한 최종 버전입니다.

* **Axial Z, Coronal Y, Sagittal X 슬라이더:** 이 3개의 슬라이더를 마우스로 움직여 병변의 3차원 중심부로 이동하며, 내부 결절이나 섭취 분포를 층별로 세밀하게 탐색할 수 있습니다.
* **시각적 신뢰도 확보:** 노란색 컨투어(ROI)가 화면에 표시됩니다. 선생님들께서 팍스(PACS) 시스템을 보며 수동으로 그렸을 때의 윤곽과, 대시보드가 제시하는 윤곽이 일치하는지 확인해 주십시오.
* **체적(ml) 확인:** 각 영상 하단에 계산된 타겟 체적(Vol: 00.00 ml)이 표시됩니다. 이 값이 임상적 직관과 부합한다면, 앞으로의 번거로운 수동 분할 작업은 이 알고리즘이 완전히 대체할 수 있습니다.

## 💡 3. 핵/입자 물리학자 들과의 연동 (Next Step)

선생님들께서 `Threshold %` 슬라이더를 통해 임상적으로 가장 타당하다고 확인해 주신 설정값(%)과 생성된 윤곽선 기하 정보는 자동으로 추출됩니다.

이 데이터는 저희 물리학자들으로 전달되어, **GEANT4, OpenTopas, OpenGATE, PHITS 몬테카를로 시뮬레이션의 Voxelized Source 입력값**으로 변환됩니다. 즉, 선생님들의 임상적 판단이 가장 정밀한 선량(Gy) 계산 엔진의 핵심 기준으로 작동하게 됩니다.

사용 중 슬라이더 반응이 느리거나, 특정 환자의 DICOM 포맷이 읽히지 않는 오류(`RuntimeError: Exception thrown in SimpleITK...`)가 발생할 경우 언제든 물리학자들에게 피드백 주시길 바랍니다.
